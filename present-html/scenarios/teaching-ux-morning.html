UX Design 101 Section 02
Methodology

The Design Thinking Process

1

Empathize

Understand users through observation, interviews, and immersion in their context

2

Define

Synthesize research into clear problem statements and user need articulations

3

Ideate

Generate a wide range of creative solutions without premature judgment

4

Prototype

Build quick, low-fidelity representations to explore ideas tangibly

5

Test

Validate with real users, gather feedback, iterate on what you learn

Based on Stanford d.school methodology — iterative, not linear

UX Design 101 Section 03
Research

User Research Methods

Qualitative
User Interviews — Deep 1-on-1 conversations
Contextual Inquiry — Observe in natural environment
Diary Studies — Longitudinal self-reporting
Focus Groups — Group discussion dynamics
Quantitative
Surveys — Large-scale data collection
A/B Testing — Compare variant performance
Analytics — Behavioral data patterns
Usability Testing — Task completion metrics
"The best research combines both — qualitative for the why, quantitative for the what."
UX Design 101 Section 04
Structure

Information Architecture

The structural design of shared information environments — how content is organized, labeled, and navigated.

Organization

Grouping content into meaningful categories and hierarchies

Labeling

Choosing clear, consistent terms that users understand

Navigation

Designing pathways that help users find what they need

Search

Enabling discovery beyond hierarchical browsing

Methods: Card sorting, tree testing, site maps, and user flow diagrams are core IA tools.
UX Design 101 Section 05
Craft

Wireframing vs. Prototyping

Wireframes
Static layout blueprints
Focus on structure, not style
Low fidelity, fast iteration
Grayscale, no real content
Prototypes
Interactive simulations
Test real user flows
Low to high fidelity range
Clickable, animated

Start with wireframes for speed, prototype when you need to test interactions

WCAG Principles (POUR)
Perceivable — Content must be presentable in ways all users can perceive
Operable — Interface components must be navigable and usable
Understandable — Content and operation must be comprehensible
Robust — Content must work across diverse technologies
"Accessible design is not a feature — it is a quality standard."
UX Design 101 Section 08
Case Study

E-Commerce Checkout Redesign

A mid-size e-commerce platform suffered from a 72% cart abandonment rate. User research revealed friction in their 5-step checkout process.

What Changed:
Reduced checkout from 5 steps to 2
Added guest checkout option
Simplified form fields from 23 to 8
Added progress indicator and trust signals
