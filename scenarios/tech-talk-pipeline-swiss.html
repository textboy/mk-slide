Flink SQL

Fraud Detection Flink SQL
SELECT user_id, COUNT(*) AS tx_count, SUM(amount) AS total_amount, TUMBLE_START(event_time, INTERVAL '5' MINUTE) FROM transactions GROUP BY user_id, TUMBLE(event_time, INTERVAL '5' MINUTE) HAVING COUNT(*) > 20 OR SUM(amount) > 10000

5-minute tumbling windows detect anomalous transaction patterns. Alerts fire in under 1 second.

06
Lessons

Five Hard-Won Insights

01
Schema Registry from day one. Use Avro + Schema Registry before production.
02
Start with Flink SQL. Java only when SQL can't express your logic.
03
Monitor consumer lag. Lag alerts prevented more incidents than code reviews.
04
Backpressure is a feature. Instrument it, tune it, respect it.
05
Exactly-once is a spectrum. Know your consistency needs before over-engineering.
