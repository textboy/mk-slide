1 研究背景

研究背景与动机

代码审查是软件质量保障的核心环节，但传统人工审查面临效率和一致性的双重挑战。

  • ● 审查耗时长 — 工业界统计表明，一次 Pull Request 的平均审查时间为 4.2 小时，大型项目中审查积压已成为开发瓶颈
  • ● 人工审查一致性低 — 不同审查者对同一段代码的反馈差异率高达 38%，审查质量受个人经验和精力状态影响显著
  • ● 现有工具的局限 — 传统静态分析工具（SonarQube、ESLint）仅覆盖规则匹配层面，无法理解代码语义、设计意图及上下文逻辑

* 数据来源：SmartBear, "2024 State of Code Review", N=3,200; Microsoft Research, "Code Reviewer Recommendations", ICSE 2023

5 贡献

主要贡献

01

领域自适应微调策略

提出基于项目编码规范和历史审查记录的两阶段微调方法，使模型能够学习特定团队的审查偏好和代码风格，在领域适配后 F1 值提升 6.7%

02

上下文感知的审查框架

设计基于 AST 依赖关系的上下文选择算法，在有限的 token 窗口内最大化相关代码覆盖率，解决跨文件审查理解的关键瓶颈

03

大规模代码审查数据集

构建并开源 CodeReviewNet 数据集，包含 12,847 个经标注的审查样本（来自 1,463 个开源项目），为后续研究提供基准

6 展望

不足与展望

当前局限

  • ◆ 多语言支持有限，目前仅覆盖 Java、Python 和 TypeScript
  • ◆ 长文件（>500 行）场景下性能下降约 15%
  • ◆ 对架构级别的设计问题识别能力仍然不足

未来方向

  • ● 增量学习机制 — 从审查者反馈中持续优化
  • ● IDE 深度集成 — 编码阶段实时审查建议
  • ● 多语言扩展 — 引入语言无关的中间表示

相关成果已发表于 ICSE 2026 Workshop (1篇) 及 IEEE TSE (在审)
