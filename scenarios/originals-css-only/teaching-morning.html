The Frontend Brief Concepts

Core Concept

What Is CSS Grid?

CSS Grid is a two-dimensional layout system. Unlike Flexbox (which works in one direction at a time), Grid lets you control both rows and columns simultaneously.

The Spreadsheet Analogy Think of Grid like a spreadsheet: you define the columns and rows, then place items into specific cells. Just like Excel, you can merge cells, skip cells, and resize columns — all with a few lines of CSS.

The Frontend Brief Concepts

Fundamentals

The 4 Building Blocks

1

Container

The parent element with display: grid. This is your spreadsheet.

2

Tracks

Rows and columns defined with grid-template-columns and grid-template-rows.

3

The fr Unit

A fraction of available space. 1fr 2fr means "1 part + 2 parts" of whatever room is left.

4

Gap

The gutter between cells. One property: gap: 1rem. No more margin gymnastics.

The Frontend Brief Process

Step by Step

How to Build a Grid Layout

Five steps, each one line of CSS.

1

Set the container

display: grid

2

Define columns

grid-template-columns

3

Define rows

grid-template-rows

4

Add gaps

gap: 1rem

5

Place items

grid-column / grid-row

Editor's Note Steps 3 and 5 are optional. Grid auto-places items by default, and rows are created implicitly as needed. Start simple!
The Frontend Brief Code

Implementation

Let's Build It

style.css CSS 
/* Step 1: Activate Grid */
.page {
  display: grid;

  /* Step 2: 3 columns — sidebar, main, aside */
  grid-template-columns: 200px 1fr 200px;

  /* Step 3: 2 rows — header + content */
  grid-template-rows: auto 1fr;

  /* Step 4: Spacing */
  gap: 1rem;

  height: 100vh;
}

/* Step 5: Place the header across all columns */
.header {
  grid-column: 1 / -1;  /* first to last line */
}

The Frontend Brief Best Practices

Analysis

Common Mistakes vs. Best Practices

Don't Do This

  • ✗Using grid-template-columns: 200px 200px 200px — breaks on small screens
  • ✗Adding margins to grid children for spacing instead of gap
  • ✗Using Grid for a single row of items (Flexbox is simpler)
  • ✗Nesting grids 3+ levels deep

Do This Instead

  • ✓Use repeat(auto-fit, minmax(250px, 1fr)) for responsive grids
  • ✓Always use gap for gutters — it respects the grid geometry
  • ✓Use Grid for page layout, Flexbox for component internals
  • ✓Try grid-template-areas for readable layouts
The Frontend Brief Summary

Key Takeaways

What to Remember

  • 1. Grid = 2D, Flexbox = 1D. If you need to control both rows and columns, Grid is your tool.
  • 2. The fr unit gives you proportional sizing without calculating percentages.
  • 3. repeat(auto-fit, minmax()) is the secret to responsive grids — no media queries needed.
  • 4. grid-template-areas makes complex layouts readable by naming regions instead of counting lines.
  • 5. Start simple: display: grid + grid-template-columns + gap gets you 80% of the way.
